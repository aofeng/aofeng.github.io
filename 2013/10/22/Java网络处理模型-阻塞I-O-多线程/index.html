Java网络处理模型-阻塞I/O+多线程

2013-10-22
  • io
  • java
  • 线程

网络服务器在处理客户端连接时，有多种不同的处理模型。JDK1.4以前，Java只有阻塞I/O，最常用的一种处理模型就是：每接收一个连接，就需要新建一个线程去处理。其处理过程如下：

  1. 接收到一个新的Socket连接。
  2. 新建一个线程处理Socket。
    1. 读取请求数据。
    2. 进行业务处理。
    3. 写响应数据。
  3. 关闭Socket。
  4. 销毁线程。

这样的网络处理模型比较简单，容易实现。但每次接收到新的连接都要新建一个线程，处理完成后销毁线程，代价大。当有大量地短连接出现时，性能比较低。
